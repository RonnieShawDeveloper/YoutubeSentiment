YouTube Sentiment Analysis

Setup & Deployment Guide

Project Overview

Welcome to the YouTube Sentiment Analysis project! This is an Angular-based web application designed to analyze the sentiment of comments on YouTube videos. It leverages Google's Gemini AI for in-depth analysis and provides a detailed report to content creators.

Technology Stack

The application is built with a modern technology stack, ensuring a robust and scalable solution:

  • Angular (v20.1): A powerful framework for building dynamic single-page applications.
  • Firebase: A comprehensive platform from Google that provides backend services like Authentication, Firestore, Remote Config, and Hosting.
  • PrimeNG & Angular Material: UI component libraries used to build a rich user interface.
  • Tailwind CSS (v4): A utility-first CSS framework for creating custom designs rapidly.

1. Firebase Project Setup

To get the application running, you first need to set up a Firebase project and the necessary API keys.

Step 1: Create a Firebase Project

  1. Go to the Firebase Console.
  2. Click on "Add project" and give your project a name.
  3. Follow the on-screen instructions to create the project. This will also create a corresponding project in the Google Cloud Console.

Step 2: Set Up Firebase Authentication

  1. In the Firebase Console, go to the Authentication section.
  2. Click "Get started".
  3. In the Sign-in method tab, select and enable Email/Password.

Step 3: Set Up Firestore Database

  1. Go to the Firestore Database section and click "Create database".
  2. Start in Production mode and select a location.
  3. Go to the Rules tab and replace the default rules with the following:
rules_version = '2';
service cloud.firestore {
        match /databases/{database}/documents {
    match /users/{userId} {
        allow read, write: if request.auth != null && request.auth.uid == userId;

        match /reports/{reportId} {
        allow read, write: if request.auth != null && request.auth.uid == userId;
      }
    }

    match /{document=**} {
          allow read, write: if false;
    }
  }
}
  1. Click Publish to save the rules.

Step 4: Obtain API Keys

This application requires two API keys: one for Google's Gemini AI and one for the YouTube Data API.

A. Get Gemini AI API Key

  1. Go to Google AI Studio.
  2. Sign in with the same Google account used for your Firebase project.
  3. Click on "Get API key" in the top left corner.
  4. In the dialog that appears, click "Create API key in new project".
  5. A new Google Cloud project will be created for you, and your API key will be displayed. Copy this key and save it somewhere safe.

B. Get YouTube Data API v3 Key

  1. Go to the Google Cloud Console.
  2. At the top of the page, select the Google Cloud project that was created with your Firebase project (it should have the same name).
  3. In the navigation menu (hamburger icon), go to APIs & Services > Library.
  4. Search for "YouTube Data API v3" and click on it.
  5. Click the Enable button.
  6. Once enabled, go to APIs & Services > Credentials.
  7. Click "+ CREATE CREDENTIALS" at the top and select "API key".
  8. A new API key will be created. Copy this key and save it. It is recommended to restrict this key to only be used by the YouTube Data API v3 for security.

Step 5: Set Up Remote Config

  1. Go to the Remote Config section in the Firebase Console.
  2. Click "Create configuration".
  3. Add two parameters using the keys you just obtained:
    • Parameter name: gemini_2_5_flash_key, Value: Your Gemini AI API Key.
    • Parameter name: youtube_api_key, Value: Your YouTube Data API v3 key.
  4. Click Publish changes.

Step 6: Set Up Firebase Hosting

  1. Go to the Hosting section.
  2. Click "Get started" and proceed through the setup wizard. We will configure the details later using the CLI.

2. Local Environment Setup

Connect your local Angular project to your new Firebase project.

  1. In the Firebase Console, go to Project Settings > General tab.
  2. Scroll to "Your apps" and click the Web icon (</>).
  3. Register the app and copy the provided Firebase SDK configuration snippet.
  4. Open src/environments/environment.ts and src/environments/environment.prod.ts.
  5. Paste the Firebase config into the firebase property in both files.
export const environment = {
  production: false,
          firebase: {
            apiKey: "AIza...",
              authDomain: "your-project-id.firebaseapp.com",
              // ... rest of your config
            },
  // ...
};

3. Building and Deploying the Application

With the project configured, you are now ready to build and deploy it.

Step 1: Install and Log in to Firebase CLI

If you don't have it, install the Firebase CLI globally. Then log in.

npm install -g firebase-tools
firebase login

Step 2: Initialize Firebase in the Project

In the project's root directory, run firebase init and answer the prompts:

  • Select Hosting.
  • Choose to Use an existing project and select your project.
  • For the public directory, enter: dist/YouTubeSentiment/browser
  • Configure as a single-page app? Yes.
  • Set up automatic builds with GitHub? No.

Step 3: Build the Application

Create a production-ready build of the application.

ng build --configuration production

Step 4: Deploy to Firebase Hosting

Deploy the built application to Firebase.

firebase deploy

Once finished, the CLI will provide the URL to your live application. Congratulations!
